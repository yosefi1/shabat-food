📋 רשימת פעולות נדרשות לפני השקה

הפריטים הבאים אינם ניתנים לתיקון בקוד — נדרשת פעולה עסקית, משפטית, או בנוגע לספקים חיצוניים.

  • 🔴
    ביקורת נגישות על ידי מורשה נגישות (ICTA) — חובה לפי חוק שוויון זכויות לאנשים עם מוגבלות (תשנ"ח-1998). יש לקבל תעודת נגישות לפני פרסום.
  • 🔴
    ביקורת משפטית של כל דפי המשפטי — מדיניות פרטיות, תנאי שימוש, מדיניות ביטולים. עורך דין מומחה בדיני אינטרנט ומסחר ישראלי.
  • 🔴
    מילוי כל [סמן כאן] בדפי המשפטי — מספר ח.פ / ע.מ, כתובת עסק, תקופות שמירת נתונים, תנאי ביטול מדויקים.
  • 🔴
    חיבור שער תשלומים (Grow / Meshulam) — יש לרשום חשבון עסקי, לקבל API credentials, ולעדכן frame-src ב-CSP לדומיין שער התשלומים.
  • 🔴
    הגדרת כל משתני הסביבה ב-Vercel — SMTP_HOST, SMTP_USER, SMTP_PASS, EMAIL_FROM, EMAIL_BUSINESS, NEXT_PUBLIC_BASE_URL, ORDERS_FILE_PATH (אם רלוונטי).
  • 🔴
    הגדרת מחירים בתפריט — כל המחירים מוצגים כ-"??" כרגע. יש לעדכן data/menuData.ts עם המחירים הנכונים.
  • 🟡
    בדיקת E2E מלאה — הזמנת ניסיון מלאה: טופס → API → שמירה → אימייל → אישור. בדיקה על iOS Safari ו-Android Chrome.
  • 🟡
    שדרוג Rate Limiting ל-Upstash Redis — הגרסה הנוכחית (in-memory) לא עובדת על serverless (Vercel). נדרש לפני עומס אמיתי.
  • 🟡
    הוספת CAPTCHA לטופס הזמנה — hCaptcha או Cloudflare Turnstile — למניעת הזמנות בוטים.
  • 🟡
    פתרון אחסון הזמנות לסביבת Vercel — ORDERS_FILE_PATH לא עובד על serverless. הצעה: Supabase (free tier), Google Sheets API, או Vercel Postgres.
  • 🟢
    הוספת SPF/DKIM/DMARC לדומיין האימייל — למניעת נפילת אימיילי אישור לספאם.
  • 🟢
    ניטור שגיאות בהפקה — חיבור Sentry (free tier) לקליטת exceptions בזמן אמת.
